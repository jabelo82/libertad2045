0. CONSTITUCIÓN DEL SISTEMA
--------------------------------------------------------------------------------

Los 7 artículos que rigen el diseño, operación y evolución de LIBERTAD_2045.
Documento completo: 01.LIBERTAD_2045_CONSTITUCIÓN.txt

  I.   DISCIPLINA          Las decisiones de trading no se alteran manualmente
                           durante la ejecución. El operador no interviene en la
                           lógica de decisión durante el ciclo operativo.

  II.  SUPERVIVENCIA       La supervivencia del sistema tiene prioridad sobre la
                           maximización del beneficio. Toda operación debe estar
                           protegida por mecanismos de control de riesgo.

  III. EVOLUCIÓN           El sistema puede evolucionar, pero de forma controlada.
       CONTROLADA          Los módulos no se modifican parcialmente — cuando se
                           introduce una mejora, el módulo completo se reemplaza.

  IV.  SIMPLICIDAD         La arquitectura debe mantenerse simple y legible.
                           Cada módulo tiene una única responsabilidad.
                           La complejidad innecesaria es una amenaza.

  V.   ROBUSTEZ            El sistema debe resistir fallos técnicos y operativos:
                           errores de datos, fallos en activos individuales,
                           interrupciones de servicios externos, reinicios.

  VI.  REGISTRO            Todo evento relevante queda documentado de forma
                           persistente. El comportamiento del sistema debe poder
                           reconstruirse años después de su ejecución.

  VII. AUTOMATIZACIÓN      La ejecución del proceso de inversión es completamente
                           automática. La intervención humana se limita a
                           mantenimiento técnico, supervisión y evolución.

--------------------------------------------------------------------------------
1. QUÉ ES EL SISTEMA
--------------------------------------------------------------------------------

LIBERTAD_2045 es un sistema de trading algorítmico autónomo diseñado para
operar en mercados de renta variable (S&P 500) de forma sistemática, sin
intervención humana en las decisiones de entrada/salida.

Creador y operador: Javier Beneito
Inicio del proyecto: 2026

Filosofía:
  - No predice el mercado; identifica tendencias y gestiona riesgo.
  - Acepta pérdidas frecuentes si las ganancias las superan con consistencia.
  - Diseñado para operar durante años, no semanas.

Estrategia base:
  - Momentum + pullback sobre la SMA50 en tendencia alcista (SMA50 > SMA200).
  - Ranking de señales por score = (close − SMA50) / ATR
                                  + pendiente_SMA200_5d / ATR  [v2]
  - Stop dinámico ATR-percentil (B1), break-even automático a +1.5×ATR.

3. ARQUITECTURA DE MÓDULOS
--------------------------------------------------------------------------------

NÚCLEO OPERATIVO

libertad2045.py       Orquestador principal. Controla el ciclo diario completo:
                      conexión → riesgo → datos → señales → ranking →
                      ejecución → rebalanceo → log → Telegram → dashboard.

conexion_ib.py        Conexión con Interactive Brokers (puerto 4002 PAPER,
                      4001 LIVE). Reintentos automáticos (3×, 5s intervalo).

data_loader.py        Descarga datos históricos (2Y, ventana ampliada en v2).
                      Calcula SMA50, SMA200, ATR, ATR_PERCENTIL.
                      Activa stop B1 al disponer de suficiente histórico ATR.

signal_engine.py      Detección de señales de entrada:
                      - Tendencia: SMA50 > SMA200
                      - Pullback adaptativo: close_prev < SMA50_prev − ATR×0.75
                        (umbral ATR-adaptativo, mejora M5 v2)
                      - Ventana de pullback: 3 días (mejora M2 v2)

position_size.py      Tamaño de posición basado en riesgo fijo:
                      riesgo por trade = 0.85% del capital
                      shares = (capital × 0.0085) / (ATR × multiplicador_dinámico)
                      multiplicador_dinámico: 2.2–4.0 según ATR_PERCENTIL.

portfolio_manager.py  Gestión de posiciones abiertas: lectura desde IBKR,
                      evaluación de stops por cierre, detección de posiciones
                      sin stop GTC activo (alerta Telegram).

order_manager.py      Gestión de órdenes pendientes. Cancela órdenes DAY,
                      nunca toca órdenes GTC (stops activos).

trade_executor.py     Ejecución de órdenes BUY STOP + SELL STOP GTC vía API.
                      Listener on_exec() para confirmar fills reales (TRADE_FILLED).

risk_guardian.py      Control de riesgo previo a cualquier operación:
                      1. Conexión IBKR activa
                      2. Ventana horaria (21:00–23:00 CET)
                      3. Capital mínimo >= 2.000 €
                      4. Drawdown <= 10% desde pico (capital_peak.txt)
                      5. Apalancamiento <= 1.0× (sin margen)

rebalance.py          Rebalanceo y gestión de stops diarios:
                      - Amplía posición si infradimensionada > 25%
                      - Reduce posición si sobredimensionada > 25%
                      - Break-even: precio >= entry + 1.5×ATR
                                  → stop sube a entry + 0.5×ATR
                      - Protección hard: posición > 25% capital → reducir.

logger.py             Registro histórico en CSV diarios (logs/LIBERTAD_YYYY-MM-DD.csv).
                      Campos: timestamp, level, event, symbol, score, shares,
                      entry, stop. Limpieza automática > 90 días.

telegram.py           Notificaciones push al operador. send_telegram() para
                      mensajes normales, send_telegram_critical() con reintentos.

universe_sp500.py     Universo de ~420 acciones del S&P 500 operables.
                      Validación de duplicados y tickers inválidos.

process_guard.py      Lock file (/tmp/libertad2045.lock) para evitar ejecuciones
                      simultáneas. acquire_lock() al inicio, release_lock()
                      en finally (siempre se libera, incluso con error).

dashboard.py          Dashboard web (dashboard.html) con posiciones, trades,
                      PnL actual en € y %, métricas del sistema.
                      Regenerado al final de cada ejecución del bot.

github_publisher.py   Publica el dashboard en GitHub Pages tras cada ejecución.

watchdog.py           Monitor de salud (12:00 L–V). 5 comprobaciones:
                      1. Heartbeat (last_run.txt < 25h)
                      2. IB Gateway accesible (puerto 4002)
                      3. Órdenes GTC activas sin cancelaciones inesperadas
                      4. RTC wakeup programado
                      5. Auto-relaunch del bot si heartbeat falla.

HERRAMIENTAS DE ANÁLISIS (no operativas, uso en Fase 3)

backtest_expandido.py    Motor principal de backtesting con reinversión anual.
                         Usado para validar mejoras antes de activarlas.
backtest_2005/10/15.py   Backtests por período.
market_filter.py         Filtro macro por estado del mercado (ALCISTA/NEUTRO/
                         BAJISTA basado en SPY). Preparado para Fase 3.
data_manager.py          Caché de datos históricos de Yahoo Finance para backtests.
                         Descarga una vez y guarda en data/. No lo usa el bot en
                         vivo, que obtiene los datos de IBKR vía data_loader.py.
montecarlo_analysis.py   Análisis Montecarlo de resultados.
generar_reportes.py      Generación de informes HTML.

4. PARÁMETROS OPERATIVOS CLAVE
--------------------------------------------------------------------------------

  Riesgo por trade    :  0.85% del capital
  Multiplicador ATR   :  dinámico 2.2–4.0 según ATR_PERCENTIL
  MAX_POS             :  10   (posiciones simultáneas máximas)
  Stop dinámico       :  B1   (ATR percentil, ventana 2Y)
  Break-even          :  +1.5×ATR entrada → stop a entry + 0.5×ATR
  Ventana datos       :  2Y   (ampliada en v2 para activar B1)
  Universo            :  ~420 acciones S&P 500
  Pullback umbral     :  close_prev < SMA50_prev − ATR_prev × 0.75
  Pullback ventana    :  últimos 3 días
  Score ranking       :  (close − SMA50)/ATR + pendiente_SMA200_5d/ATR
  Capital mínimo      :  2.000 €
  Drawdown máximo     :  10% desde pico

    22:00             →  RTC despierta la torre (ciclo noche)

    22:20             →  trading-night-shutdown: programa RTC 11:50 + apaga

    ~/.config/systemd/user/ibgateway.service
    Restart=on-failure, RestartSec=60
    Arranca automáticamente en cada boot vía ibc_autostart.sh

  Ficheros de ejecución:
    run_bot.sh        →  activa venv, lanza libertad2045.py
    watchdog.sh       →  lanza watchdog.py
    setup.sh          →  instalación inicial del entorno (reinstalación)
    ibc_autostart.sh  →  arranca Xvfb + IB Gateway vía IBC
